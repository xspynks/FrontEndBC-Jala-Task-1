Nem 8 nem 80, Nem 80, Nem 80

Um artigo que encontrei utilizando o Daily.dev me chamou atenção. Apesar do clickbait, o autor se aprofunda no assunto e questiona se realmente nunca devemos repetir o nosso código. No entanto, o artigo não é um endosso à repetições de código, mas sim uma reflexão sobre o que é DRY e como podemos aplicá-lo de forma mais eficiente.

Motivos para repetir seu código:

  • Avoid unecessary abstraction;
  • Creates abstraction on-demand;
  • Increase productivity;
  • Avoid major refacts when business logic changes;
  • Code is easier to read;
  • Code is easier to understand;
  • Code is easier to maintain;
  • Code is easier to test;
  • Code is easier to debug;
  • Code is easier to extend;
  • Code is easier to reuse;
  • Code is easier to refactor;
  • You can always refact (and you will, at least in most cases).

Extensão útil

Daily.dev é uma extensão para os principais navegadores modernos, muito útil para se manter atualizado em sua área de atuação dentro da tecnologia, especialmente se você é um Desenvolvedor. Feita com ReactJS + NextJS, ela faz uma curadoria das principais notícias e artigos de acordo com os seus assuntos preferidos.

Conclusão

  1. Tente não repetir o seu código, mas quando perceber que é o melhor caminho, vá em frente.
  2. Se você não sabe se deve repetir ou não, provavelmente não deve repetir.
  3. Se repetir e depois de algum tempo sentir a necessidade de criar uma abstração, refatore.
  4. Instale a Daily.dev e mantenha-se atualizado.
